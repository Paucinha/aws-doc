Laboratório - Gerenciamento de recursos com marcação.

Este laboratório é dividido em duas partes:

  • Na parte Tarefa deste laboratório, você usará a CLI (Interface de linhas de comando) da AWS para inspecionar as tags atribuídas a várias instâncias do Amazon EC2. Em seguida, você usará scripts pré-fornecidos para desativar e iniciar várias instâncias do Amazon EC2 simultaneamente, com base em suas tags.
  • Na parte Desafio deste laboratório, você terá que detectar uma maneira de encerrar instâncias que não implementam tags específicas.

Objetivos Após concluir este laboratório, você poderá:

  • Aplicar tags a recursos existentes da AWS.
  • Encontrar recursos com base em tags.
  • Usar a CLI da AWS ou o AWS SDK para PHP para interromper e encerrar as instâncias do Amazon EC2 com base em determinados atributos do recurso.

Duração

Este laboratório levará aproximadamente 45 minutos para ser concluído.

Cenário

O ambiente deste laboratório (ilustrado abaixo) consiste em:

  • Amazon VPC denominada Lab VPC
  • Sub-rede pública
  • Sub-rede privada
  • Instância Linux do Amazon EC2 chamada CommandHost [As ferramentas da Interface de linhas de comando (CLI) da AWS foram pré-instaladas e configuradas para você nesta instância]
  • Oito Instâncias do Amazon EC2 do Linux
  • As instâncias privadas têm três tags personalizadas aplicadas a elas:

Na parte Tarefa deste laboratório, você fará login no host de comando e executará alguns comandos para encontrar e alterar a tag Version em todas as instâncias de desenvolvimento. Você executará vários exemplos que mostram como usar a sintaxe do JMESPath compatível com a opção --query da CLI da AWS para retornar uma saída altamente formatada. Em seguida, você usará um conjunto de scripts pré-fornecidos para parar e reiniciar todas as instâncias marcadas como pertencentes ao ambiente de desenvolvimento.

Como encontrar instâncias de desenvolvimento para o projeto

Agora que você está conectado, você pode usar a CLI da AWS para encontrar os recursos em sua sub-rede privada que pertencem ao projeto ERPSystem e estão no ambiente denominado desenvolvimento. Você também verá como usar a opção --query da CLI da AWS para produzir resultados altamente formatados.

  1. Para encontrar todas as instâncias em sua conta que estão marcadas com uma tag Project (Projeto) e um valor ERPSystem, copie o seguinte comando e execute-o na janela do terminal do Linux:

    xxxxxxxxxx
    aws ec2 describe-instances --filter "Name=tag:Project,Values=ERPSystem"

    O comando deve gerar o conjunto completo de parâmetros disponíveis para as sete instâncias marcadas com Project=ERPSystem. Isso gera muitos resultados, e a maior parte não se aplica a esse laboratório. Na próxima etapa, você usará o parâmetro --query para refinar os resultados.

  2. Use o parâmetro --query para limitar a saída do comando anterior a apenas o ID da instância descoberta:

    xxxxxxxxxx
    aws ec2 describe-instances --filter "Name=tag:Project,Values=ERPSystem" --query 'Reservations[*].Instances[*].InstanceId'

    Suas entradas de saída agora consistirão em uma lista de IDs de instância:

    [ [ "i-135b491e" ], [ "i-3e584a33" ], … ]

    O comando --query usado neste exemplo usa a sintaxe curinga JMESPath para especificar que o comando deve percorrer todas as reservas e todas as instâncias e retornar o InstanceId para cada instância nos resultados de retorno. Essa é uma melhoria em relação ao retorno de todas as propriedades de nossas instâncias. Mas e se você quiser incluir vários campos na saída?

  3. Copie o comando a seguir e execute-o na janela do terminal do Linux para incluir o ID da instância e a zona de disponibilidade de cada instância no resultado retornado:

    xxxxxxxxxx
    aws ec2 describe-instances --filter "Name=tag:Project,Values=ERPSystem" --query 'Reservations[*].Instances[*].{ID:InstanceId,AZ:Placement.AvailabilityZone}'

    Dois pares nome/valor são retornados para cada resultado. Este comando baseia-se no uso da sintaxe JMESPath pelo comando anterior usando chaves para especificar uma consulta de várias propriedades em cada instância retornada:

    object.{Alias1:PropertyName1,Alias2:PropertyName2,[…]}

    Como podemos observar, você pode especificar um alias para cada propriedade a fim retornar um formato de saída mais abreviado.

    Com essa saída, você pode ver claramente que seu filtro funcionou, com a exibição apenas das instâncias associadas ao projeto ERPSystem. No entanto, você provavelmente não será capaz de identificar quais instâncias estão sendo retornadas com base nessas informações. Nas próximas etapas, você verá como incluir o valor de suas tags personalizadas na saída de retorno.

  4. Para incluir o valor da tag Project (Projeto) na saída, copie e execute o seguinte comando no terminal do Linux:

    xxxxxxxxxx
    aws ec2 describe-instances --filter "Name=tag:Project,Values=ERPSystem" --query 'Reservations[*].Instances[*].{ID:InstanceId,AZ:Placement.AvailabilityZone,Project:Tags[?Key==`Project`] | [0].Value}'

    Sua saída agora inclui o valor da tag Project (Projeto):

    [[{ "Project": "ERPSystem", "AZ": "us-west-2a", "ID": "i-3250b838" }], … ]

    O valor de uma tag nomeada específica pode ser recuperado por meio de uma consulta JMESPath usando a seguinte sintaxe:

    xxxxxxxxxx
    Tags[?Key==\`Project\`] | [0].Value

    Essa sintaxe instrui o JMESPath a encontrar todos os elementos no array Tags cujo valor de Key (Chave) é Project (Porjeto). A saída desse comando (que será um único elemento de Tags) é direcionada para outro comando, que seleciona a primeira instância desse conjunto filtrado e seleciona o parâmetro nomeado Value (Valor), que é o valor da tag Project (Projeto). Esse resultado é atribuído ao alias Project (Projeto).

  5. Copie e execute o seguinte comando para incluir também as tags Environment e Version na saída:

    xxxxxxxxxx
    aws ec2 describe-instances --filter "Name=tag:Project,Values=ERPSystem" --query 'Reservations[*].Instances[*].{ID:InstanceId,AZ:Placement.AvailabilityZone,Project:Tags[?Key==`Project`] | [0].Value,Environment:Tags[?Key==`Environment`] | [0].Value,Version:Tags[?Key==`Version`] | [0].Value}'

    Os resultados fornecerão uma imagem mais completa das instâncias atualmente associadas ao projeto chamado ERPSystem:

    [[{ "Environment": "production", "Project": "ERPSystem", "Version": "1.0", "AZ": "us-west-2a", "ID": "i-3250b838" }], … ]

  6. Por fim, adicione um segundo filtro de tag para ver apenas as instâncias associadas ao projeto denominado ERPSystem que pertencem ao ambiente development (desenvolvimento):

    xxxxxxxxxx
    aws ec2 describe-instances --filter "Name=tag:Project,Values=ERPSystem" "Name=tag:Environment,Values=development" --query 'Reservations[*].Instances[*].{ID:InstanceId,AZ:Placement.AvailabilityZone,Project:Tags[?Key==`Project`] | [0].Value,Environment:Tags[?Key==`Environment`] | [0].Value,Version:Tags[?Key==`Version`] | [0].Value}'

    Você deve ver apenas duas instâncias retornadas por esse comando, ambas com um valor de tag Project (Projeto) de ERPSystem e um valor de tag Environment (Ambiente) de desenvolvimento:

    [[{ "Environment": "development", "Project": "ERPSystem", "Version": "1.0", "AZ": "us-west-2a", "ID": "i-9552ba9f" }], … ]

 

Alteração da tag Version (Versão) para o processo de desenvolvimento

Neste procedimento, você alterará todas as tags Version (Versão) nas instâncias marcadas como development (desenvolvimento) para o projeto ERPSystem.

Você pode definir individualmente essas propriedades em cada instância afetada, mas uma abordagem automatizada é mais prática. Você pode usar um script de shell Bash do Linux simples para criar com base nas consultas criadas anteriormente e modificar entradas de tag como uma operação em lote.

  1. No CommandHost, abra o arquivo /home/ec2-user/change-resource-tags.sh:

    xxxxxxxxxx
    nano change-resource-tags.sh

  2. Examine o conteúdo do script:

    ​x
    #!/bin/bash
    

    ids=$(aws ec2 describe-instances --filter "Name=tag:Project,Values=ERPSystem" "Name=tag:Environment,Values=development" --query 'Reservations[*].Instances[*].InstanceId' --output text)
    

    aws ec2 create-tags --resources $ids --tags 'Key=Version,Value=1.1'

    Esse script primeiro usa o comando aws ec2 describe-instances para retornar apenas uma lista de IDs de instância das máquinas de desenvolvimento pertencentes ao projeto ERPSystem. Em seguida, ele passa esses valores para o comando aws ec2 create-tags, que cria uma nova tag ou (neste caso) substitui uma tag existente.

    Observe como o primeiro comando usa a opção --output text para manipular o resultado retornado como texto em vez de JSON. O uso desse comando em vez do JSON em um resultado retornado simples (neste caso, uma lista de IDs) pode facilitar a manipulação do resultado retornado e passá-lo para outros comandos.

  3. Feche o editor nano e execute este comando no prompt de comando do Linux:

    xxxxxxxxxx
    ./change-resource-tags.sh

  4. Para verificar se o número da versão nessas instâncias foi incrementado e se outras caixas de não desenvolvimento no projeto ERPSystem não foram afetadas, copie e execute o seguinte comando:

    xxxxxxxxxx
    aws ec2 describe-instances --filter "Name=tag:Project,Values=ERPSystem" --query 'Reservations[*].Instances[*].{ID:InstanceId, AZ:Placement.AvailabilityZone, Project:Tags[?Key==`Project`] |[0].Value,Environment:Tags[?Key==`Environment`] | [0].Value,Version:Tags[?Key==`Version`] | [0].Value}'

 

Tarefa 2: Interromper e iniciar recursos por uma tag

Nesta tarefa, você usará um script pré-fornecido para interromper e iniciar um conjunto de instâncias marcadas como instâncias de desenvolvimento.

 

Examinar o script Stopinator

  1. Na instância de host de comando, cd no diretório aws-tools no diretório inicial:

    xxxxxxxxxx
    cd aws-tools

  2. Abra o arquivo stopinator.php e examine seu conteúdo:

    xxxxxxxxxx
    nano stopinator.php

    O script stopinator.php é um script simples que usa o AWS SDK para PHP a fim de interromper e reiniciar instâncias com base em um conjunto de tags. Isso permite cenários como desligar servidores de ambiente de desenvolvimento no final do dia e reiniciá-los na manhã seguinte. O script procurará instâncias que correspondam às tags especificadas em todas as regiões da AWS. O script recebe os seguintes argumentos:

    • -t: um conjunto de tags no seguinte formato: name=value;name=value O script converte essas tags no formato esperado pela chamada Ec2::DescribeInstance() do AWS PHP. Se esse parâmetro opcional estiver ausente, o script identificará e encerrará todas as instâncias do Amazon EC2 em execução na conta.
    • -s: um parâmetro booleano; nenhum argumento é necessário. Quando este parâmetro está presente, as instâncias identificadas por -t são iniciadas em vez de interrompidas.

  3. Saia do editor nano.

 

Interromper e reiniciar o processo de desenvolvimento do ERPProject

Nesta tarefa, você usará o script stopinator.php para ativar e ativar seu ambiente de desenvolvimento para o projeto ERPSystem.

  1. No shell do Linux, execute o script stopinator.php:

    xxxxxxxxxx
    ./stopinator.php -t"Project=ERPSystem;Environment=development"

    A saída deve ser assim, indicando que duas instâncias serão interrompidas na região atual da AWS. (Os resultados serão diferentes dependendo da região em que o laboratório está sendo executado.)

    Region is us-east-1 (Região é us-east-1) No instances to stop in region (Não há instâncias a serem interrompidas na região) Region is us-west-1 (Região é us-west-1) No instances to stop in region (Não há instâncias a serem interrompidas na região) Region is us-west-2 (Região é us-west-2) Identified instance i-9552ba9f (Instância encontrada i-9552ba9f ) Identified instance i-d35fb7d9 (Instância encontrada i-d35fb7d9) Stopping all identified instances... (Interrompendo todas as instâncias identificadas...) […] No instances to stop in region (Não há instâncias a serem interrompidas na região) Region is sa-east-1 (Região é sa-east-1) No instances to stop in region (Não há instâncias a serem interrompidas na região)

  2. No menu Services (Serviços), clique em EC2.

  3. No painel de navegação, clique em Instances (Instâncias).

  4. Verifique se duas instâncias estão sendo interrompidas ou já foram interrompidas.

  5. Volte para a sessão SSH para Host de comando e, no prompt do Linux, reinicie suas instâncias com o seguinte comando:

    xxxxxxxxxx
    ./stopinator.php -t"Project=ERPSystem;Environment=development" -s

  6. Volte para a janela do Console de gerenciamento do EC2 e verifique se as duas instâncias que foram desativadas anteriormente estão sendo reiniciadas.

 

Tarefa 3: Desafio: Encerrar instâncias que não estejam em conformidade

Neste desafio, será solicitado que você detecte uma maneira de encerrar instâncias que não estejam em conformidade com determinadas diretrizes de segurança.

Observação Se você já estiver familiarizado com a AWS, recomendamos que tente esse desafio por conta própria usando as informações fornecidas nesta seção antes de ler a solução detalhada fornecida na próxima seção. Quando tiver concluído o desafio, verifique seu trabalho analisando a solução detalhada.

Descrição do desafio Cenário: sua empresa quer que você crie processos automatizados que encerrarão automaticamente instâncias que possam permitir uma possível violação de segurança. Você identificou uma lista de riscos de segurança e agora está decidindo como implementá-los de forma eficiente usando comandos da CLI da AWS ou o SDK do PHP para AWS. Desafio: sua primeira tarefa de segurança é simples: encontrar todas as instâncias em sua sub-rede privada que não implementam a tag Environment (Ambiente) e encerre-as (ou seja, uma política de "tag-or-terminate").

Dicas:

  • Se você não estiver confortável com PHP ou uma linguagem de programação semelhante (como Python ou Ruby) para a qual um SDK da AWS esteja disponível, tente usar uma série de comandos da CLI da AWS para executar essa tarefa.
  • A chamada para PHP da AWS Ec2:: terminateInstances () pode encerrar instâncias. O comando equivalente da CLI da AWS é aws ec2 terminate-instances.
  • Você pode usar o script stopinator da seção 2 como referência para qualquer código que escrever.

Visão geral da solução do desafio

Há várias maneiras de abordar esse problema usando uma variedade de soluções programáticas ou de linha de comando. A solução geral para o problema consiste nas seguintes etapas:

  • Identificar todas as instâncias que atualmente têm a tag Environment (Ambiente) definida.
  • Comparar isso com a lista de todas as instâncias disponíveis e registrar os IDs de instância de todas as instâncias que não fazem parte da lista obtida da Tarefa 1.
  • Fornecer os IDs das instâncias não marcadas à AWS usando o comando aws ec2 stop-instance (CLI da AWS) ou a chamada de API Ec2:: terminateInstances () (PHP).

A solução a seguir demonstra como esse problema pode ser resolvido usando um script PHP.

  3. Este bloco de código usa o método describeInstances (), um filtro para encontrar todas as instâncias que têm a tag Environment (Ambiente) definida, independentemente do valor da tag. Ele armazena todos os IDs de instância que encontra em uma tabela de hash.

  4. Examine o segundo bloco do código.

Esse código examina todas as instâncias em sua sub-rede e as compara com a lista de instâncias marcadas com a tag Environment (Ambiente). Se uma instância não estiver na lista marcada, seu ID de instância será adicionado a uma lista de instâncias a serem encerradas.

  1. Examine a última seção do script.

Essas linhas usam a lista de IDs de instância não compatíveis como um argumento para o método terminateInstances.
